# SEO Content Structure

How to structure articles for search engines.

---

## The Anatomy of Rankable Content

```
[META: Title Tag + Meta Description]
│
[H1: Title]
│
[Introduction: Hook + Promise + Keyword]
│
├── [H2: Section 1]
│   ├── [H3: Subsection] (if needed)
│   └── [Content]
│
├── [H2: Section 2]
│   ├── [H3: Subsection]
│   └── [Content]
│
├── [H2: Section N...]
│
├── [H2: FAQ/Related Questions] (optional)
│
└── [Conclusion: Summary + CTA]
```

---

## Page Elements

### Title Tag (Meta Title)

**Purpose:** What shows in search results

**Requirements:**
- Primary keyword included
- Under 60 characters (55-60 ideal)
- Compelling/clickable
- Matches search intent

**Formula Options:**
- `[Primary Keyword]: [Benefit/Qualifier]`
- `[Number] [Type] for [Keyword] ([Year])`
- `How to [Keyword] [Without Pain/In Timeframe]`
- `[Keyword] vs [Keyword]: [Value Add]`

**Examples:**
- "Email Marketing Strategy: 7 Tactics That Actually Work (2024)"
- "How to Write Headlines That Convert (Without Sounding Clickbait)"
- "Notion vs Obsidian: Which Note-Taking App Wins?"

---

### Meta Description

**Purpose:** The description in search results (influences CTR)

**Requirements:**
- 150-160 characters
- Include primary keyword
- Clear value proposition
- Call to action or curiosity hook

**Formula:**
`[What they'll learn/get]. [Key benefit/differentiator]. [Action or hook].`

**Example:**
"Learn the exact email strategy that grew our list 10x in 6 months. No paid ads, no cold outreach. Get the full framework."

---

### H1 (Page Title)

**Purpose:** Main headline on the page

**Requirements:**
- One H1 per page
- Primary keyword included
- Can be longer than title tag
- Hooks the reader to continue

**Relationship to Title Tag:**
- Can be identical
- Can be expanded version
- Should be consistent in topic

---

### Introduction

**Purpose:** Hook reader, set expectations, include keyword

**Structure:**

```
[Hook: Question, statistic, bold statement, or pain point]

[Context: Brief setup of the problem/topic]

[Promise: What they'll learn/gain from reading]

[Roadmap (optional): Quick preview of sections]
```

**Requirements:**
- Primary keyword in first 100 words
- Short paragraphs (1-3 sentences)
- Hooks immediately—no throat clearing
- States clear benefit of reading

**Length:** 100-200 words

---

### Body Sections (H2s)

**Purpose:** Main content organized logically

**H2 Guidelines:**
- Each H2 covers one main point
- Include secondary keywords where natural
- Should make sense as table of contents
- 300-500 words per section typically

**H3 Guidelines:**
- Break up long sections
- Sub-points within main point
- Not required for short sections

**Content Guidelines:**
- Short paragraphs (2-4 sentences)
- Use lists for multiple items
- Include examples and specifics
- Add relevant images/media

---

### FAQ Section (Optional)

**Purpose:** Answer "People Also Ask" questions, capture featured snippets

**When to Include:**
- Related questions exist in PAA
- Space to add value
- Questions are truly asked

**Structure:**
```
## Frequently Asked Questions

### [Question matching PAA]
[Direct, concise answer in first 1-2 sentences]
[Expanded context if helpful]

### [Next question]
[Answer]
```

---

### Conclusion

**Purpose:** Summarize, reinforce, direct next action

**Structure:**
```
[Summary: Key takeaways in 1-2 sentences]

[Action: What they should do now]

[CTA: Related content, email signup, or product]
```

**Length:** 50-150 words

## Content Format by Intent

### How-To Guides

```
H1: How to [Achieve Goal]
Intro: Promise + outcome
H2: Step 1: [First action]
H2: Step 2: [Second action]
H2: Step N...
H2: Common Mistakes (optional)
Conclusion: Summary + next steps
```

### List Posts

```
H1: [Number] [Type] for [Goal/Audience]
Intro: Why this list matters
H2: 1. [Item]
H2: 2. [Item]
H2: N...
Conclusion: How to choose/next steps
```

### Comparison Posts

```
H1: [A] vs [B]: [Value Add]
Intro: Who should read, what you'll learn
H2: Quick Comparison (table)
H2: [A] Overview
H2: [B] Overview
H2: Key Differences
H2: Which Should You Choose?
Conclusion: Recommendation
```

### Comprehensive Guides

```
H1: The Complete Guide to [Topic]
Intro: Scope + who it's for
H2: What Is [Topic]?
H2: Why [Topic] Matters
H2: How to [Core Process]
H2: [Advanced Considerations]
H2: Common Mistakes
H2: FAQ
Conclusion: Summary + resources
```

## Word Count Guidelines

| Content Type | Typical Range | Notes |
|--------------|---------------|-------|
| How-to (simple) | 1,000-1,500 | Get to the point |
| How-to (complex) | 2,000-3,500 | Cover thoroughly |
| List post | 1,500-3,000 | Depends on list size |
| Comparison | 2,000-3,500 | Need adequate depth |
| Comprehensive guide | 3,000-5,000+ | Aim to be definitive |

**Rule:** Match competitor length, then add unique value—don't pad.

## Internal Linking Structure

**Minimum:** 3-5 internal links per article

**Link Placement:**
- Introduction: Link to related foundational content
- Body: Link to detailed posts on subtopics
- Conclusion: Link to next logical content

**Anchor Text:**
- Descriptive (not "click here")
- Natural to sentence
- Keyword-relevant when honest

---

## Image Optimization

**Alt Text:**
- Describe the image
- Include keyword if genuinely relevant
- Keep under 125 characters

**File Names:**
- Descriptive: `email-marketing-funnel.png`
- Not generic: `image001.png`

**Placement:**
- Break up long text sections
- Support/illustrate points
- Featured image matches topic
